# Momentum — Backend

A local-first productivity AI that reads your Chrome tabs, scores your focus
with Gemma (via Ollama), and surfaces what you're working on.

**No API keys. No cloud. Everything runs on your machine.**

---

## Prerequisites

| Tool | Install |
|------|---------|
| Python 3.10+ | <https://python.org> |
| Ollama | <https://ollama.com/download> |
| Google Chrome | <https://google.com/chrome> |

## 1 — Install Ollama & pull Gemma

```bash
# Install Ollama (macOS — other platforms: https://ollama.com/download)
brew install ollama

# Pull the Gemma 3 model
ollama pull gemma3

# Start the Ollama server (keep this terminal open)
ollama serve
```

Ollama will be available at `http://localhost:11434`.

## 2 — Launch Chrome with remote debugging

You must start Chrome with the `--remote-debugging-port` flag so the
tab reader can connect via Chrome DevTools Protocol (CDP).

```bash
# macOS
open -a "Google Chrome" --args --remote-debugging-port=9222

# Linux
google-chrome --remote-debugging-port=9222

# Windows (PowerShell)
& "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222
```

> **Tip:** Close all Chrome windows first, then run the command above
> so Chrome starts fresh with debugging enabled.

Verify it works: open <http://localhost:9222/json> — you should see a
JSON array of your open tabs.

## 3 — Install Python dependencies

```bash
cd backend
pip install -r requirements.txt
```

## 4 — Start the backend

```bash
cd backend
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

The API is now at `http://localhost:8000`.
Check health: <http://localhost:8000/health>

## 5 — Run the tab reader

In a separate terminal:

```bash
cd backend
python tab_reader.py          # loops every 15 seconds
python tab_reader.py --once   # single scan
```

The reader will:
1. Connect to Chrome at `localhost:9222`
2. Extract titles, URLs, and page text from every open tab
3. POST the data to `http://localhost:8000/tabs`
4. Print the focus score and insight to the terminal

---

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/tabs` | Submit tab data → returns insight + focus score |
| `GET` | `/focus` | Latest focus score (polled by frontend every 15s) |
| `GET` | `/sessions` | 10 most recent sessions |
| `GET` | `/memory` | Memory summary from the latest session |
| `GET` | `/health` | Server status and model name |

### POST /tabs — request body

```json
{
  "tabs": [
    { "title": "Page Title", "url": "https://example.com", "text": "page content..." }
  ]
}
```

### POST /tabs — response

```json
{
  "success": true,
  "focus_score": 72,
  "focus_label": "Getting scattered",
  "insight": {
    "working_on": "Researching FastAPI middleware patterns",
    "next_action": "Finish reading the Starlette middleware docs and implement the auth layer",
    "stuck_signal": null
  },
  "tab_count": 8
}
```

## Focus Score

The focus score is an integer from **0 to 100** generated by Gemma based on:

- **Number of tabs** — fewer related tabs = higher score
- **Tab relatedness** — are tabs about the same project/topic?
- **Distraction sites** — social media, news, video platforms lower the score
- **Clear primary task** — is there an obvious thing you're working on?

### Labels

| Score Range | Label | Meaning |
|-------------|-------|---------|
| 70–100 | **Deep focus** | You're locked in on one task |
| 40–69 | **Getting scattered** | Multiple threads, some drift |
| 0–39 | **Highly fragmented** | Tab chaos, no clear direction |

```
│  (port 9222) │              │    .py        │                 │  (port 8000) │
└──────────────┘              └──────────────┘                 └──────┬───────┘
                                                                      │
                                                        ┌─────────────┼──────────────┐
                                                        │             │              │
                                                        ▼             ▼              ▼
                                                   ┌─────────┐  ┌─────────┐  ┌───────────┐
                                                   │  Ollama  │  │ SQLite  │  │  React    │
                                                   │ (Gemma)  │  │   DB    │  │ Frontend  │
                                                   └─────────┘  └─────────┘  └───────────┘
```

---

## No API Keys Needed

Momentum runs **entirely locally**:

- **Ollama** runs the Gemma model on your machine
- **Chrome CDP** reads tabs from your local browser
- **SQLite** stores session history in a local file
- **FastAPI** serves the API on localhost

Nothing leaves your machine. 🔒
